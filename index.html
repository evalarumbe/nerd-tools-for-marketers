An overview on caching and cookies

Every browser you have installed sets aside some temporary storage on your computer for caching and cookies. (It's common for web developers to install many different browsers to help test for inconsistencies). The main job of private / incognito windows is to ignore temporary storage and let you browse as if you had no existing cache or cookies.

Cache

A cache is a place to temporarily store files so they're faster to retrieve. We do this all the time: if you're getting ready to make dinner tonight, you might set out some ingredients on the kitchen counter just temporarily so they're faster to reach, even though their rightful home is inside the cupboard.

Computers typically use many different kinds of caches, at hardware and software level. In web development, we are usually talking about the browser cache.

The cache is useful for storing heavy files like images, on websites you visit frequently. This lets pages load faster without making you repeatedly download the same image.

Cookies

A cookie is a file stored on your computer. Websites that you visit use these to identify you on repeated visits. Cookies can help site owners answer questions like:

  • Are you logged in?
  • Have you visited this site before?
  • Which pages did you look at? If you were on the checkout page, we can infer that you were close to buying. If you got to the purchase confirmation page, you've converted.

This is why we can log into different accounts on the same website by using a private / incognito session, or a different browser.
